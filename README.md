# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Данные 

Интерфейс `IProduct`(Товар) описывает структуру данных товара в приложении. Он используется для типизации объектов товаров, которые мы получаем с сервера. 

```
interface IProduct {
  id: string;          // Уникальный идентификатор товара
  description: string; // Описание товара
  image: string;       // URL изображения товара
  title: string;       // Название товара
  category: string;    // Категория товара
  price`: number | null; // Цена товара (может отсутствовать)
}
```

Интерфейс `IBuyer`(Покупатель) описывает структуру данных покупателя, которые нам понадобятся для оформления заказа. Он используется для типизации данных формы заказа.

```
interface IBuyer {
  payment: TPayment; // Способ оплаты (два варианта: онлайн/оффлайн)
  email: string;     // Электронная почта покупателя
  phone: string;     // Номер телефона покупателя
  address: string;   // Адрес доставки
}

```
`type TPayment` = 'online' | 'offline'; // Возможные способы оплаты

### Модели данных

#### ProductList - Модель каталога товаров
Класс отвечает за хранение и управление списком товаров, которые можно купить в приложении. Кроме того, он хранит товар, который выбран для подробного отображения.

Конструктор:
`constructor(initialProducts: IProduct[] = [])` - Конструктор принимает initialProducts (необязательный) - начальный массив товаров типа IProduct[] для предварительного заполнения каталога

Поля класса:
`private _items: IProduct[]` - массив всех товаров в каталоге. Каждый элемент массива (он же товар) соответствует интерфейсу IProduct
`private _previewItem: IProduct | null` - текущий товар, выбранный для подробного отображения. Принимает значение null если ни один товар не выбран

Методы класса:
`setItems(products: IProduct[]): void` - полностью заменяет текущий массив товаров новым массивом, переданным в параметре
`getItems(): IProduct[]` - возвращает полный массив всех товаров, хранящихся в каталоге
`getItem(id: string): IProduct | undefined` - осуществляет поиск товара по id. Возвращает найденный товар или undefined если товар с указанным id не существует
`setPreviewItem(item: IProduct): void` - сохраняет переданный товар как текущий для подробного отображения
`getPreviewItem(): IProduct | null` - возвращает товар, выбранный для подробного отображения, или null если товар не выбран

#### Cart - Модель корзины
Класс отвечает за хранение товаров, выбранных пользователем для покупки, предоставляет методы для управления содержимым корзины и вычисления таких данных как, например, общая стоимость и количество товаров. 

Конструктор:
`constructor(initialItems: IProduct[] = [])` - Конструктор принимает initialItems (необязательный) - начальный массив товаров типа IProduct[] для инициализации корзины

Поля класса:
`private _items: IProduct[]` - массив товаров, добавленных в корзину. Каждый элемент массива (он же товар) соответствует интерфейсу IProduct

Методы класса:
`getItems(): IProduct[]` - возвращает массив всех товаров, находящихся в корзине
`addItem(item: IProduct): void` - добавляет новый товар в корзину. Принимает объект товара и помещает его в массив корзины
`removeItem(item: IProduct): void` - удаляет конкретный товар из корзины 
`clear(): void` - полностью очищает корзину, удаляя все товары из массива
`getTotalAmount(): number` - вычисляет и возвращает общую стоимость всех товаров в корзине (на основе значений полей price товара)
`getItemsCount(): number` - возвращает общее количество товаров в корзине (длину массива товаров)
`contains(itemId: string): boolean` - проверяет наличие товара в корзине по его id. Возвращает true если товар с указанным id присутствует в корзине, иначе - false

#### Buyer - Модель покупателя
Класс отвечает за хранение, управление и валидацию данных покупателя, которые понадобятся для оформления заказа. Обеспечивает сохранение контактных данных и способа оплаты, предоставляет механизм валидации полей с возможностью получения информации об ошибках.

Конструктор:
`constructor(initialData: Partial<IBuyer> = {})` - Конструктор принимает initialData (необязательный) - объект типа Partial<IBuyer> для начальной инициализации данных покупателя (позволяет установить только часть полей)

Поля класса:
`private _payment: TPayment | null` - выбранный способ оплаты. Может принимать значения 'cash' (наличные), 'card' (карта) или null если способ не выбран
`private _email: string` - адрес электронной почты покупателя. Строка может быть пустой если email не указан
`private _phone: string` - номер телефона покупателя. Строка может быть пустой если телефон не указан
`private _address: string` - адрес доставки заказа. Строка может быть пустой если адрес не указан

Методы класса:
`setData(data: Partial<IBuyer>): void` - обновляет данные покупателя. Принимает объект с частичными данными и обновляет только указанные поля, сохраняя значения остальных полей
`setPayment(payment: TPayment): void` - устанавливает способ оплаты. Принимает значение типа TPayment и сохраняет его в соответствующем поле
`setEmail(email: string): void` - устанавливает адрес электронной почты. Принимает строку и сохраняет её в соответствующем поле
`setPhone(phone: string): void` - устанавливает номер телефона. Принимает строку и сохраняет её в соответствующем поле
`setAddress(address: string): void` - устанавливает адрес доставки. Принимает строку и сохраняет её в соответствующем поле
`getData(): IBuyer | null` - возвращает все данные покупателя в виде объекта IBuyer. Если какое-либо обязательное поле не заполнено, возвращает null
`clear(): void` - полностью очищает все данные покупателя, сбрасывая все поля к пустым значениям

`validate(): ValidationResult` - выполняет полную валидацию всех полей покупателя. Возвращает объект типа ValidationResult, содержащий только те поля, в которых обнаружены ошибки
`validatePayment(): string | null` - проверяет валидность поля способа оплаты. Возвращает строку с текстом ошибки если поле невалидно, или null если ошибок нет
`validateEmail(): string | null` - проверяет валидность поля email. Возвращает строку с текстом ошибки если поле невалидно, или null если ошибок нет
`validatePhone(): string | null` - проверяет валидность поля телефона. Возвращает строку с текстом ошибки если поле невалидно, или null если ошибок нет
`validateAddress(): string | null` - проверяет валидность поля адреса. Возвращает строку с текстом ошибки если поле невалидно, или null если ошибок нет

`type ValidationResult`= {
  payment?: string;  // Текст ошибки валидации способа оплаты, если поле невалидно
  email?: string;    // Текст ошибки валидации email, если поле невалидно
  phone?: string;    // Текст ошибки валидации телефона, если поле невалидно
  address?: string;  // Текст ошибки валидации адреса, если поле невалидно
}
Объект содержит свойства только для тех полей, которые не прошли валидацию. Если поле валидно, соответствующее свойство отсутствует в объекте. Например, объект {payment: 'Не выбран вид оплаты', email: 'Укажите email'} указывает на ошибки в полях оплаты и email, при этом поля телефона и адреса считаются валидным

### Слой коммуникации

#### Класс WebLarekAPI 
Класс является представителем коммуникационного слоя приложения и отвечает за взаимодействие с сервером "Веб-ларёк". Реализует методы для получения товаров из каталога и отправки заказов. 

Конструктор:
`constructor(baseUrl: string, options: RequestInit = {})` - Конструктор принимает
baseUrl (string) - базовый URL сервера API 
options (RequestInit -  необязательный) - дополнительные опции для HTTP-запросов. По умолчанию: {}. 

Поля класса:
`private api: Api` - экземпляр базового класса Api, инкапсулирующий логику выполнения HTTP-запросов. Инициализируется в конструкторе с переданными параметрами baseUrl и options.

Методы класса:
`getProductList(): Promise<IProduct[]>`- Выполняет GET запрос на эндпоинт /product/ для получения каталога товаров.
Возвращает - Promise<IProduct[]> - промис, который разрешается массивом объектов товаров типа IProduct

`submitOrder(orderData: IOrder): Promise<IOrderResult>` - Выполняет POST запрос на эндпоинт /order/ для отправки данных заказа.
orderData (IOrder) - объект с данными заказа, должен соответствовать интерфейсу IOrder
Возвращает - Promise<IOrderResult> - промис, который разрешается объектом с результатом создания заказа IOrderResult
